How Do We Judge If a Model Is "Usable"?

📈 Benchmarks

  • MMLU — Multi-discipline knowledge
  • HumanEval — Coding ability
  • GSM8K — Math reasoning
  • SWE-bench — Real software engineering

💻 Hardware Requirements

  • Does the model fit in my GPU VRAM?
  • Is inference speed practical? (tokens/sec)
  • Can quantization make it fit smaller GPUs?

🎯 Task Suitability

  • Simple chatbot? 7B is enough
  • Code generation? 13B+ recommended
  • Complex reasoning? 70B+ or API

NVIDIA Minitron — Pruning + Distillation Combined

Achieves equivalent performance with 40x fewer tokens than training from scratch

Step 1: Teacher Correction

Fine-tune the large model (15B) on target data to align output distributions

Step 2: Pruning

Width pruning to remove neurons/heads. Physically shrink the model (15B → 8B, 4B)

Step 3: Distillation Retraining

Retrain the pruned model using teacher outputs to recover performance

40xTraining token savings · 16% MMLU improvement (vs from-scratch)

Microsoft Phi — Data Quality Beats Scale

Traditional Approach

  • Crawl trillions of internet tokens
  • Rough filtering
  • Win by volume
  • Requires massive models (70B+)

Phi Approach

  • Generate "textbook-quality" synthetic data via GPT-4
  • Strategic topic selection + knowledge gap analysis
  • Win by quality
  • Small models achieve large-model performance

📖 Key Terms Glossary

ParameterA single number storing learned knowledge
B (Billion)1 billion. 7B = 7 billion parameters
TokenSmallest unit of text the model processes
QuantizationReduce bit precision to shrink model size
DistillationTransfer large model knowledge to small model
Fine-tuningAdditional training on specific tasks
RLHFAlign model using human feedback
MoEMixture of Experts. Activate subset of params
VRAMGPU memory. Model must fit to run inference
GGUFLocal model file format (llama.cpp)
