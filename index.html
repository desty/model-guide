양자화 & 증류

거대 모델을 실용적으로 쓸 수 있게 만드는 두 가지 핵심 기술

양자화(Quantization) — 모델 다이어트

숫자의 정밀도를 낮춰서 모델 크기를 줄이는 기술

FP32 (32비트 원본)~280GB
FP16 (16비트) — 50% 절약~140GB
INT8 (8비트) — 75% 절약~70GB
INT4 (4비트) — 87.5% 절약~35GB

* 70B 모델 기준 메모리 사용량

GPTQ

GPU 추론에 최적화. 사전 양자화 방식.

GPU 전용

GGUF

CPU+GPU 혼합 실행 가능. 로컬 실행에 인기. llama.cpp 사용.

CPU+GPUQ4_K_M 추천

AWQ

중요한 가중치를 보호하며 양자화. 성능 손실 최소화.

고정밀

bitsandbytes

HuggingFace 연동. 실시간 양자화 (NF4, INT8).

HF 연동

지식 증류(Knowledge Distillation) — 큰 모델의 지식을 작은 모델에 전수

증류 vs 파인튜닝 vs 양자화

소형 모델(SLM) 만들기

큰 모델에서 출발하거나, 처음부터 똑똑하게 만들거나

소형 모델을 만드는 5가지 방법

flowchart TB
    BIG["큰 모델이 이미 있다"] --> PATH1["1. 지식 증류\n큰 모델 출력을 따라 배움"]
    BIG --> PATH2["2. 가지치기 Pruning\n덜 중요한 뉴런/레이어 제거"]
    BIG --> PATH3["3. 증류+가지치기 결합\nMinitron 방식"]
    SCRATCH["처음부터 만든다"] --> PATH4["4. 고품질 데이터 전략\nPhi 방식"]
    SCRATCH --> PATH5["5. 파인튜닝\n기존 모델 + LoRA/QLoRA"]

가지치기(Pruning) — 모델을 깎아내기

Width Pruning (너비)

레이어를 "얇게" — 뉴런/어텐션 헤드 수 줄임

NVIDIA Minitron — 가지치기 + 증류의 결합

처음부터 학습하는 것 대비 40배 적은 토큰으로 동급 성능

1단계: 교사 보정

큰 모델(15B)을 타겟 데이터로 미세조정하여 출력 분포를 맞춤

2단계: 가지치기

Width Pruning으로 뉴런/헤드 제거. 물리적으로 모델 크기를 줄임 (15B → 8B, 4B)

3단계: 증류 재학습

교사 모델의 출력으로 가지치기된 모델을 재학습하여 성능 회복

40x 학습 토큰 절약 · MMLU 16% 향상 (vs 처음부터 학습)

Microsoft Phi — 데이터 품질이 규모를 이긴다

전통적 접근법

  • 인터넷 크롤링 수조 토큰
  • 대충 필터링
  • 양으로 승부
  • 거대 모델(70B+) 필요

Phi 접근법

  • GPT-4로 "교과서급" 합성 데이터 생성
  • 전략적 주제 선정 + 지식 갭 분석
  • 질로 승부
  • 소형 모델로도 대형급 성능
